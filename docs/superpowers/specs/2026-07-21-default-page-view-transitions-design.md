# View Transition между страницами по умолчанию

## Цель

Автоматически добавлять нативное затухание при обычной навигации между страницами сайта из области действия расширения без отдельной пользовательской настройки.

## Архитектура

- Подключить к content script отдельный CSS-модуль с `@view-transition { navigation: auto; }`.
- Использовать стандартную анимацию браузера без перехвата ссылок и изменения логики навигации Bitrix.
- Загружать CSS на всех страницах, уже охваченных content script: `https://cnc1.ru/*` и `https://www.cnc1.ru/*`.

## Поведение и ограничения

- Переход применяется только при навигации между документами одного origin, когда обе страницы поддерживают и разрешают View Transition.
- Переход между `cnc1.ru` и `www.cnc1.ru` остаётся обычным, поскольку это разные origins.
- Неподдерживающие браузеры игнорируют правило и продолжают обычную навигацию.
- При `prefers-reduced-motion: reduce` анимация отключается.
- Переключатель в панели не добавляется: возможность включена по умолчанию вместе с расширением.

## Проверка

- Production-сборка расширения проходит после изменения.
- CSS присутствует в собранном content script или связанном с ним ресурсе.
- Переход по обычной внутренней ссылке одного origin даёт стандартное затухание в поддерживающем браузере.
- Навигация не перехватывается JavaScript и работает как раньше при отсутствии поддержки API.
