# Всегда включённый поиск по select

## Цель

Убрать переключатель «Поиск по select» из пользовательских настроек и запускать помощник поиска для больших `select` на каждой подходящей странице.

## Изменения

- Удалить `selectHelperEnabled` из публичной модели настроек, значений по умолчанию и списка улучшений боковой панели.
- Удалить соответствующий пункт и хук хранения из устаревшего popup-списка функций.
- В отложенной инициализации всегда вызывать `selectHelper.injectButtons()`; сохранённое ранее значение `selectHelperEnabled: false` не должно отключать функцию.
- Не удалять старый ключ из `chrome.storage.local`: он станет безвредным устаревшим значением, а отказ от миграционной записи уменьшит объём и риск изменения.

## Поведение и совместимость

После обновления расширения поиск по большим спискам включается автоматически. Остальные переключатели и их значения работают без изменений. Повторная инициализация остаётся безопасной благодаря существующей реализации `selectHelper`.

## Проверка

- TypeScript-проверка проходит без ссылок на удалённое поле.
- Production-сборка расширения проходит.
- В панели отсутствует строка «Поиск по select».
- На поддерживаемой странице кнопки поиска появляются независимо от ранее сохранённого значения настройки.
